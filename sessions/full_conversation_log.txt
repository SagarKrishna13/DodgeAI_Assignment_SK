### 2. Configuration Layer

Implement config.py:

Implement:

1. db/connection.py:

* SQLite connection
* row_factory for dict-like access

* Dynamically extract schema
* List tables + columns + types
* Return formatted schema string

Constraints:

* No hardcoding
* Fully dynamic

Implement ingestion module:

* Read all JSONL files recursively

* Flatten nested JSON

Constraints:

* No hardcoded file names
* Replace tables if exist

Implement schema_graph.py:

### 6. Dynamic Subgraph (Core Feature)

Implement subgraph.py:

Function:
fetch_subgraph(entity_ids)

Behavior:

* Fetch only relevant rows
* Traverse relationships (Order → Delivery → Invoice → Payment)
* Return nodes + edges

Constraints:

* No full dataset loading
* Efficient queries

### 7. LLM Prompt (Join Intelligence + Guardrails)

Implement system prompt:

Include:

* Schema injection
* Domain restriction
* Join dictionary
* SQL generation rules

Rules:

* Only SELECT
* Always LIMIT
* Use exact schema columns
* Wrap SQL in <sql> tags

Enhancements:

* SQLite compatibility
* Avoid hallucination
* Use correct table-column mapping

---

### 8. Query Engine (End-to-End Pipeline)

Implement query_engine.py:

Flow:

1. Build messages (system + history + query)
2. Call LLM
3. Extract SQL from <sql> tags
4. Validate SQL
5. Execute query
6. Return rows
7. Call LLM again for explanation
8. Extract entity IDs from SQL results

Return:

* answer
* sql
* rows
* highlighted_ids

---

Implement sql_validator.py:

* Use sqlparse
* Allow only SELECT
* Reject unsafe queries
* Enforce LIMIT
* Reject multiple statements
